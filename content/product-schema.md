# Product Schema for Individual Detail Pages

Use the following structure for every product page:

1. Product Name
2. Family
3. Subcategory
4. Short industrial overview
5. Core applications
6. Typical specifications
7. Available grades / forms
8. Technical notes
9. Documentation support (CoA, TDS, MSDS)
10. Recommended usage / buyer guidance
11. Product image file reference

Suggested fields for a spreadsheet or JSON source:

- product_name
- family
- subcategory
- product_slug
- overview
- applications
- typical_specifications
- grades_forms
- technical_notes
- documentation
- recommended_use
- image_file
